# Challenge and Adaptation Report — PIG-001

## 1. What the Theory Brief proposes (restated fairly)

THEORY_001 consolidates six theory-side reports into one handoff. It finds
Pigment a coherent Atlas-first product with a materially implemented
accountless Taste layer, and directs a **bounded stabilization and evidence
pass, not a redesign**, under the frozen objective: *let a first-time
visitor discover art, admire works, form and recover a local Taste
Passport, understand how their actions affect it, and recognize Pigment as
an atlas of relationships in art*. It freezes the Atlas-first hierarchy,
preserves the multi-door homepage as rollback baseline, defers navigation
restructuring to an Atlas Coherence Pass, excludes accounts/social/new
taste math, defines ten workstreams (A–J), 35 acceptance criteria in three
priority families with a release rule, a thirteen-artifact evidence packet,
frozen terminology, and a list of decisions reserved for MIRA/the product
owner. It requests, as Claude's first output, a neutral feasibility and
scope classification (§0, six items). This report is that output.

## 3. What we challenge

**CH-1 [critical] The acceptance set cannot be passed as written.**
Criteria 10–11 demand source/type/confidence on influence edges and
item-level provenance/rights on images; no schema holds any such field
(js/influences.js three-tuples; js/catalog-1.js `image:{src,page,status}`
with `page` an English Wikipedia article). Criterion 12 ("no indispensable
provenance or legal unknown is hidden") is a negative existential no finite
evidence can establish. The Release rule blocks `human_review_ready` on any
unsupported P0 criterion — so the gate as written is permanently closed.
— **adapt**: rewrite per §5.3.

**CH-2 [critical] Workstream I is a second project presented as an evidence
gate.** Aggregate coverage counts over the corpus plus "visible
differentiation between fact, editorial interpretation, legend, disputed
claim, and unknown" is schema design + migration + audit + new UI in
sequence. The honest bounded version exists: a **rights register** over the
actually rendered image surface — measured at **783 images** (255 catalog
`pd` + 528 artworks.js gallery works; data-integrity-report.md Q2), ~80–90%
automatable because the fetch pipeline already receives and discards the
Commons `extmetadata` it needs. Container recommendation (Seurat, Q1):
**hybrid** — out-of-band register now (no build authorization required),
minimal `image.license` + `image.commons` fields folded into the schema in
the authorized build phase, because PIGMENT.md §14 wants attribution close
to the record and two sources of truth will drift. A 10-image pilot
(~1–1.5h) prices the rest before commitment. Note recorded in D-008: the
783 figure supersedes both earlier internal framings of the surface's size.
— **adapt**: split I into (a) rights register [in scope], (b) claim-level
sourcing [separate task, gated on release positioning], (c) uncertainty UI
[deferred to Atlas Coherence Pass].
